# AI Usage Policy

Maintainer Signal Kit may use AI-assisted development, but every public claim
must remain human-reviewable and evidence-backed.

## Allowed Uses

- drafting tests and documentation
- summarizing issue or pull request queues
- preparing release notes from verified commits
- reviewing generated evidence packs for missing caveats
- checking whether public claims are supported by repository evidence

## Required Human Review

Maintainers must review AI-assisted changes before merge. In particular, review:

- generated application text
- public adoption or ecosystem-impact claims
- security-sensitive code paths
- redaction behavior
- network behavior

## Not Allowed

- inventing users, stars, downloads, endorsements, sponsors, or contributors
- hiding that the project is new
- adding fake testimonials or fake production usage
- submitting private data to third-party systems without maintainer approval

## Submission Standard

If a sentence cannot be supported by public repository evidence, it should be
phrased as a plan, a goal, or a placeholder requiring maintainer verification.
